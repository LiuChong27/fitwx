# 微信小程序审核处理说明

## 这次被拒的直接原因

当前版本包含完整的用户内容发布与公开展示能力，代码里已经存在这些社交/笔记特征：

- `pages/grid/discover.vue`
  用户可发布训练笔记、查看公开内容、评论、分享、举报。
- `pages/grid/components/publish-modal.vue`
  用户可填写文本内容并上传图片。
- `pages/chat/chat.vue`
  用户之间可私聊，并支持发送图片。
- `uniCloud-aliyun/cloudfunctions/fit-discover-api/index.js`
  后端提供笔记发布、评论、点赞、分享链路。
- `pages/admin/feed-review.vue`
  已有内容审核台。
- `pages/admin/report-review.vue`
  已有举报处理台。

这和微信审核给出的结论是一致的：这是带 UGC 公开展示能力的产品，按当前形态应补充 `社交-笔记` 类目。

## 最省钱的处理方案

不要硬删功能，直接按 `社交-笔记` 重新提审。

建议你在微信公众平台这样填写：

1. 补充服务类目：`社交-笔记`
2. 功能页面说明：
   - `pages/grid/discover`：用户发布训练笔记，内容包括文字、图片、地点和训练标签，公开展示给其他用户查看。
   - `pages/chat/chat`：用户在约练前进行一对一沟通。
   - `pages/report/create`：用户可对违规内容进行举报。
3. 内容安全说明：
   - 文本内容发布前做前端拦截。
   - 云端对笔记、评论、邀约、举报说明做内容安全检测。
   - 后台有人工审核页和举报处理页。
4. 测试路径建议：
   - 首页进入 `笔记` tab
   - 点击右上角 `发布`
   - 发布一条训练笔记
   - 打开笔记详情
   - 评论/举报

本小程序主要提供约练和训练笔记服务。用户可发布训练笔记，内容包含文字、图片、地点和训练标签，其他用户可公开查看、评论、举报，并在约练前通过私信沟通。平台已配置内容安全检测、人工审核和举报处理机制。
